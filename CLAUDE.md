# Learner_AI
### 项目总览
本项目旨在实现一个 LLM 驱动的学习应用, 需要实现以下功能:
1. 用户询问某一学科的知识如何学习, 应用可以制定详细且可执行的学习计划.
2. 总结用户的知识网络, 实现可交互的知识网络和基于网络的 AI 问答.
3. 应用驱使用户主动学习.
#### 与现有的 LLM app 的差异点
现有的 LLM app 已经可以做到制定学习计划, 但是交互性不强, 而且比较浅显地止于读什么教材的水平, 只能给出一个清单式的计划, 我们希望 AI 实现的计划是可以通过应用增强可执行性, 提供包括但不限于以下的功能:
1. 可交互的计划书, 应用提醒用户学习, 进行学习检测
2. 用户学到的东西实时更新到知识网络
3. AI 提供更有深度的计划
对于 **更有深度的计划**, 以用户现在提出 “我要学习数学分析” 为例, 给出以下模版:
```
第一部分: 极限
Day 1 数列极限的 epsilon-delta 定义, 动机和理解
Day 2 简单的数列极限证明
Day 3 数列极限的性质
Day 4 函数的极限
Day 5 证明练习
...
```
具体不详细列举. 我们同时希望 AI 可以给出讲义而不是让用户自己啃教材.
### 实现路径
为了让 AI 可以给出高质量的计划和讲义, AI 需要以下的 tool:
1. Web search
这是为了让 AI 在网络上查询资料
2. PDF read / Image read
我们希望 AI 生成的内容可以参考高质量的教材, 它们可能以 pdf 或者图片的形式给出.
3. Agent team
### 交互功能
1. 计划阶段的查看与提醒
2. 知识网络
3. AI 对话
### 框架
LLM 调用 DeepSeek API, 识图和搜索在实现过程中商量.
整个项目使用 Electron 实现交互. 使用 TypeScript. 
### 要求
强调不同模块解耦, Render 层和后端不能混在一起. 强调可拓展性.
以上的内容比较简略, 在实现过程中有任何稍微不确定的东西都要和我商讨确定.

### 知识网络 UI 实现约定
知识网络页面当前采用接近 Obsidian Local Graph 的交互方式, 后续修改需要保持以下方向:

1. 数据链路不能破坏: 已有学习计划需要能自动生成知识网络. 新建计划保存时写入知识节点和边, 图谱为空时从已有计划补初始化, 这类逻辑属于 main / persistence / IPC 层, 不应为了 UI 效果挪到 renderer.
2. Renderer 只负责展示和交互: `src/renderer/src/pages/KnowledgeGraphPage.tsx` 可以做布局、筛选、高亮、视图聚焦, 但不要直接修改图谱持久化数据. 图谱节点和边仍通过 `window.learnerAI.graph.get()` 与 `getNodeDetail()` 读取.
3. 视觉风格要克制: 深色画布、小球节点、细线连接、少量动画. 避免大面积渐变、花哨图例、过强光晕和高饱和多色主题. 节点默认以灰蓝色为主, 焦点和搜索命中可轻微提亮.
4. 全局布局要关系优先: 初始布局不应随机散点. 先根据有向边从根节点分层, 再用轻量力学收紧; 相连节点应更近, 不同连通分量可以分开摆放. 根节点定义为入度为 0 的节点, 如果没有入度为 0 的节点, 选择连接度最高的节点作为兜底根节点.
5. 焦点布局要动态: 选中节点后, 该节点成为局部视图焦点. 父节点靠左, 子节点靠右, 二阶/三阶节点靠近对应邻居, 更远节点保留全局相对方向和远近并适度压缩, 不要把远节点投影成以焦点为圆心的圆环.
6. 视图聚焦要平滑但不过度: 切换焦点时可以自动居中并轻微缩放, 但不要放得太大导致上下文不可读. 焦点边可更亮, 远边可降低存在感, 但远节点仍应可见.
7. 标签策略要上下文感知: 不要全量显示标签. 焦点节点、一阶相连节点、搜索命中节点常显标签; 鼠标附近节点临时显示标签; 远节点默认隐藏标签, 以避免文字拥挤.
8. 鼠标交互可以有轻微响应: 鼠标附近节点可以小幅向光标移动并显示标签, 但不要改变底层图谱数据或破坏当前焦点布局.
9. 搜索模式优先级最高: 搜索时应临时使用全局布局, 不套当前焦点局部布局; 匹配节点和相邻节点保留, 其他节点淡化, 并自动将命中节点 fit 到视野内. 搜索框需要避免被画布拖拽/鼠标交互干扰.
10. 右侧信息栏保持常驻: 显示当前节点信息、父节点、子节点和所有根节点. 点击父节点、子节点、根节点应切换焦点并居中.
11. 交互实现应保持轻量: 尽量用 React Flow 自带能力和局部计算, 避免引入重型图布局库或复杂物理引擎. 动态效果应该只在 renderer 内通过临时布局数据完成.
