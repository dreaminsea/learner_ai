# Learner_AI
### 项目总览
本项目旨在实现一个 LLM 驱动的学习应用, 需要实现以下功能:
1. 用户询问某一学科的知识如何学习, 应用可以制定详细且可执行的学习计划.
2. 总结用户的知识网络, 实现可交互的知识网络和基于网络的 AI 问答.
3. 应用驱使用户主动学习.
#### 与现有的 LLM app 的差异点
现有的 LLM app 已经可以做到制定学习计划, 但是交互性不强, 而且比较浅显地止于读什么教材的水平, 只能给出一个清单式的计划, 我们希望 AI 实现的计划是可以通过应用增强可执行性, 提供包括但不限于以下的功能:
1. 可交互的计划书, 应用提醒用户学习, 进行学习检测
2. 用户学到的东西实时更新到知识网络
3. AI 提供更有深度的计划
对于 **更有深度的计划**, 以用户现在提出 “我要学习数学分析” 为例, 给出以下模版:
```
第一部分: 极限
Day 1 数列极限的 epsilon-delta 定义, 动机和理解
Day 2 简单的数列极限证明
Day 3 数列极限的性质
Day 4 函数的极限
Day 5 证明练习
...
```
具体不详细列举. 我们同时希望 AI 可以给出讲义而不是让用户自己啃教材.
### 实现路径
为了让 AI 可以给出高质量的计划和讲义, AI 需要以下的 tool:
1. Web search
这是为了让 AI 在网络上查询资料
2. PDF read / Image read
我们希望 AI 生成的内容可以参考高质量的教材, 它们可能以 pdf 或者图片的形式给出.
3. Agent team
### 交互功能
1. 计划阶段的查看与提醒
2. 知识网络
3. AI 对话
### 框架
LLM 调用 DeepSeek API, 识图和搜索在实现过程中商量.
整个项目使用 Electron 实现交互. 使用 TypeScript. 
### 要求
强调不同模块解耦, Render 层和后端不能混在一起. 强调可拓展性.
以上的内容比较简略, 在实现过程中有任何稍微不确定的东西都要和我商讨确定.
